# 04 — UX

Design stance: **a production studio that feels like a reading app.** v1's UI was a
pipeline dashboard with a great canvas bolted on; v2 leads with content (cast, storyboard,
pages, reader) and hides machinery entirely. No pipeline tab. No DAG. No step buttons.

## Information architecture

```
/                      Library (projects grid)
/new                   New project wizard (3 steps)
/p/:id                 Project home = chapter rail + book overview
/p/:id/cast            Cast gallery (gate 1)
/p/:id/c/:ch           Chapter workspace (storyboard ⇄ pages tabs)
/p/:id/c/:ch/read      Reader (also public-shareable link)
/p/:id/settings        Providers, style, budget
```

## Flows

### 1. New project wizard (upload → first progress in <30s)

1. **Drop file(s)** — m4b/mp3/epub/txt. m4b chapters auto-detected; shows detected
   chapter list immediately (title + duration), editable.
2. **Pick a style** — visual grid of style-pack presets, each shown as the *same sample
   panel* rendered in that style (pre-baked assets, instant). Optional custom prompt +
   reference upload.
3. **Confirm & go** — shows estimated cost range up front ("Casting ~$0.40 · Storyboard
   ~$0.15/chapter · Full render ~$2–4/chapter"). Cost transparency before any spend —
   v1 had none.

Ingest + script for chapter 1 start immediately; the user lands on the project home
watching live progress.

### 2. Project home — the chapter rail

A vertical list of chapter cards (not a kanban — v1's board overweighted process):

```
┌─ 1. The Black Ship          ● Ready to read        [Read] [Pages]
┌─ 2. Ashes of the Port       ◐ Drawing 23/41 panels  ▓▓▓▓▓▓░░░░
┌─ 3. The Governor's Table    ⏸ Storyboard awaiting review   [Review]
┌─ 4. Nightfall               ○ Queued
```

- One status line + one action button per card. Failures show a plain-language error
  with a single [Retry] (v1 surfaced raw Effect stack traces).
- Header: book cover (first splash panel), bible peek (top characters with portraits),
  cumulative cost ticker.
- Everything updates via SSE; no manual refresh, no polling spinners that hang (v1's
  GetStatus timeout is architecturally impossible here — status is a DB read).

### 3. Gate 1 — Cast gallery

The "casting session." Grid of character cards; each shows 2–4 generated candidates +
the visual spec text.

- Actions per character: pick candidate ✓ · regenerate with a note ("older, scar on
  left cheek") · edit spec text directly · upload own image.
- Locations and key objects in secondary tabs.
- Sticky footer: `Approve cast (12/14 chosen)` — enabled when all principals resolved.
- Later chapters that introduce new faces surface a small "3 new characters need casting"
  card on the chapter rail — mini-gate, same UI, only the new entries.

### 4. Gate 2 — Storyboard review (per chapter)

Two-pane: left = scene/beat outline (collapsible tree); right = page spreads as
**wireframes**: layout rectangles with description snippets + dialogue text in place.

- Inline edit any description/dialogue (debounced autosave).
- Panel operations: split beat into two panels, merge, change layout template (visual
  template picker), change camera (shot-size chips: WIDE / MED / CLOSE / SPLASH).
- Dialogue is shown *as bubbles* in the wireframe so lettering density problems are
  visible before rendering ("this panel has 40 words — split?"). Automatic warnings on
  over-dense panels.
- Sticky footer: `Looks good — draw this chapter` with panel count + cost estimate.
- After approval the same screen becomes the render progress view: wireframe rectangles
  fill in with images as renders land (live, satisfying, and doubles as QA skim).

### 5. Chapter workspace — Pages tab (post-render editing)

The v1 canvas, rebuilt with its lessons; this was v1's best surface, so keep the shape:

- Page-by-page editor (spread view), zoom/pan preserved across operations.
- Click panel → side panel: prompt (editable), seed, QA badge + notes, render version
  strip (thumbnails of v1..vn — click to switch `active_render`), [Regenerate] with
  optional note, ref-images used.
- Bubble editing: drag/resize/retail bubbles, inline text edit, add/delete; typeset
  instantly (SVG overlay — no re-render round-trip, unlike v1's baked bubbles).
- Batch bar: "Re-render 4 QA-flagged panels" / "Re-render all panels with <character>"
  (cast-version staleness).
- Every mutation optimistic + TanStack Query invalidation; errors surface as toasts,
  never silent (v1: silent `catch(() => {})`).

### 6. The Reader (flagship)

`/p/:id/c/:ch/read` — mobile-first, consumes the publish manifest only (static data +
audio URL), so it's fast and shareable.

- **Panel-by-panel mode (default on phones)**: one panel fills the screen; audio plays;
  auto-advance on the timing map with a soft crossfade. Tap = pause; swipe = manual
  navigation (audio seeks to that panel's `startSec`).
- **Page mode (tablet/desktop)**: full page visible; current panel gets a subtle spotlight
  as narration progresses.
- Bubbles render as crisp DOM/SVG text over the image — always legible at any zoom, and
  they can appear timed with the narration for read-along emphasis.
- Transport: play/pause, speed (0.75–2×), chapter picker, scrubber with panel tick marks.
- Silent projects (text ingest): same reader without transport; tap to advance.
- Export menu here: CBZ · PDF · MP4 (motion comic) · copy share link.

## Design system

- shadcn/ui + Tailwind, dark-first (reading app), one accent color per style pack.
- Panel/page imagery is always shown with correct aspect (no stretch — v1 bug class).
- Empty states teach the flow ("Upload a book to start casting your comic").
- Error boundaries on every route segment (v1 added them last; here from scaffold).
- Accessibility: reader transport fully keyboard-driven; bubbles are real text (screen-
  reader readable — a genuine win of typeset-over-baked lettering); captions track for
  MP4 export.

## What intentionally does not exist

- Pipeline/DAG visualization, step retry buttons, cron scheduling UI.
- Global pause. Work is per-chapter; chapters never block each other.
- A separate "knowledge" tab as primary nav (bible is a peek panel + casting surface).
- Manual embedding/RAG controls. Retrieval is invisible plumbing.
